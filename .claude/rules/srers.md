# SRERS Mandate
---
alwaysApply: true
---

All code must adhere to **SRERS**: Simple, Reliable, Efficient, Robust, Straightforward.

- Prefer boring, readable code over clever abstractions
- Optimize for clarity first, performance second
- Fix root causes, not symptoms
- No bandaids, no workarounds, no silenced errors
- No unused, broken, deprecated, or commented-out code
- No backward compatibility code, polyfills, or legacy fallbacks
- Comments describe what code does NOW, not what it used to do
- No over-engineering, no speculative complexity
