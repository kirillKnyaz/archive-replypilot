# F001 — Stale tab binding TTL

## Problem

`background.js` stores `pending_tab_<N>` entries in `chrome.storage.session` when a tab is opened for enrichment. They're cleared when scraping completes (via `clearTabCtx`) or when the tab is closed (via `tabs.onRemoved`). Two cases leave stale entries:

- User closes Chrome mid-scrape → session storage clears on browser close, so not a real leak
- Maps page takes >60s to load or hangs → entry sits forever until browser close

## Fix

Add `queuedAt: Date.now()` to the ctx stored by `setTabCtx`. On service worker wake, sweep entries older than 10 minutes. Low urgency — only becomes a real problem with heavy usage.

## Files

- `browser-extension/background.js` — prepare-enrich handler, add sweep on startup
