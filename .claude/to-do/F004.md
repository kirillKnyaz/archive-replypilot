# F004 — Frontend listens for rp_extension_error

## Problem

`bridge.js` already posts `{ type: 'rp_extension_error' }` back to the page when the extension is invalidated or unreachable. Neither LeadPage nor LeadsTable listens for it. User waits the full 60s timeout before seeing any feedback.

## Fix

In `LeadPage.jsx` and `LeadsTable.jsx`, add a `useEffect` that subscribes to `window.addEventListener('message', ...)` and handles `event.data.type === 'rp_extension_error'`:
- Set an error state immediately
- Abort the polling loop
- Show a clear message: "Extension not installed or needs refresh"

## Files

- `frontend/src/pages/leads/LeadPage.jsx`
- `frontend/src/components/campaigns/LeadsTable.jsx`
