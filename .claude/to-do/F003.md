# F003 — Redact Authorization in logs

## Problem

No HTTP request logger exists yet, but when one is added (morgan, pino-http, etc.) the default format includes the `Authorization` header. API tokens would leak into logs.

## Fix

When adding the logger, configure a redactor:
- morgan: custom token that masks the header
- pino: `redact: ['req.headers.authorization']`

Defer until request logging is introduced. Note in the logger setup docs.
