# F002 — Auth DB lookup caching

## Problem

`backend/middleware/authenticate.js:25` does `prisma.user.findUnique({ where: { apiToken } })` on every request with an `rp_` token. At scale this is:
- One DB roundtrip per API call from the extension
- A DoS vector: hammering with random rp_ strings → DB load
- Unnecessary latency on the hot path

## Fix

Add in-memory LRU cache keyed by token → `{ userId, email, expiresAt }`. Short TTL (30-60s) so revoked tokens stop working quickly. Invalidate on POST /auth/token (when regenerating).

Use a simple Map with manual eviction or a lightweight package like `lru-cache`.

## Files

- `backend/middleware/authenticate.js` — add cache layer before DB lookup
- `backend/routes/auth.js` — call cache invalidation on token regenerate
