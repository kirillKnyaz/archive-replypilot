# F005 — Comment maps-scraped fire-and-forget

## Problem

`background.js` has two message listeners: `who-am-i` (returns `true` for async sendResponse) and `maps-scraped` (returns nothing). The asymmetry looks like a bug but is intentional — content.js doesn't await a response from maps-scraped.

## Fix

One-line comment in background.js above the `maps-scraped` handler explaining the pattern: fire-and-forget, no reply expected, content.js closes its side of the port as soon as the message is sent.
